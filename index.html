ⓘ Project Context

Bloom is a comprehensive digital pelvic health solution developed by Sword Health. The program delivers guided exercise sessions to women across six life stages, addressing conditions including intimacy pain, menstruation pain, and constipation. Members complete up to 16 sessions and report outcomes at each session, including pain relief and the Patient Global Impression of Change (PGIC) score.

This portal was developed as a portfolio project to demonstrate data analytics capabilities aligned with a Product Data Analyst role. The underlying database contains simulated data for 10,000 members, generated to reflect realistic clinical outcome distributions.

ⓘ Infrastructure Notes

Neon provides a permanent free tier with serverless scaling. The connection uses SSL mode require. The default database name is neondb.

Railway hosts the Metabase Docker image alongside an internal PostgreSQL instance for Metabase metadata. The Neon database is connected to Metabase via Admin settings. The Railway deployment can be paused when not in use to avoid billing.

Public dashboard embedding is enabled in Metabase under Admin settings, Embedding. The embed renders correctly only when served over HTTPS. It will not display when the HTML file is opened locally via the file:// protocol.

ⓘ Schema Overview

The Bloom database follows a star schema with two fact tables and five dimension tables. The dim_date table is a standalone calendar dimension used for time-based filtering in Metabase. It is not linked via foreign keys to other tables but is joined on the date_id column in fact_sessions.

Tables

All tables in the Bloom database.

⚠
This database contains simulated data generated for portfolio demonstration purposes. All member records, session outcomes, and satisfaction scores are fictitious.
Dimension Descriptive attributes Fact Measurable events
TableTypeColumnsDescription
dim_date Dimension 8 Calendar dimension covering 2023 to 2026. Used for time-based filtering in Metabase. Not linked via foreign keys; joined on date_id in fact_sessions.
dim_life_stage Dimension 3 Six female life stages: Young Adulthood, Pregnancy, Postpartum, Perimenopause, Menopause, and Postmenopause. Each stage reflects a distinct hormonal and pelvic health context.
dim_conditions Dimension 3 Three pelvic health conditions addressed by the Bloom program: Intimacy Pain, Menstruation Pain, and Constipation. Each member is assigned exactly one condition.
dim_locations Dimension 6 Geographic reference table for the 40 US cities represented in the member base. Stores city name, state, country, and coordinates for map visualisations in Metabase.
dim_users Dimension 9 Member profiles for 10,000 simulated Bloom users. Includes age, location, marital status, household, life stage, and assigned condition.
fact_sessions Fact 6 One row per session per member, covering up to 16 sessions. Records session number, date, pain relief flag (yes/no, first occurrence only), and PGIC score (1-7). Condition is derived via the member profile in dim_users.
fact_program_outcomes Fact 5 One row per member who completed the full 16-session program (7,500 members). Records total sessions completed, satisfaction score (0-10), and program end date.
